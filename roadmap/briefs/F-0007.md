# Builder Brief — F-0007: Per-feature authorized_paths edit guard (PreToolUse)

> Immutable. Build exactly this; report back; nothing more.

## Feature

Mechanically enforce per-feature authorized_paths / forbidden_paths on all Edit/Write operations via PreToolUse hook. Design resolves active feature selection (CLAUDE_ACTIVE_FEATURE env contract set by builder briefs), glob matching, fail-closed when active, safe no-active case for orchestrator/maintenance. This is the core engine guard enabling all product features (F-0017/F-0018 etc) to stay in scope without drift or violations.

## Spec

- PreToolUse hook (in .claude/hooks/verify-gate.sh or guard-bash.sh + TS/JS side if needed) that on Edit/Write:
  - Reads CLAUDE_ACTIVE_FEATURE env (or session marker) to identify the current feature id.
  - Loads that feature's authorized_paths + forbidden_paths from features.json (or cached).
  - For the target file path being edited/written: glob-match against authorized (allow if matches any), then check forbidden (block if matches any).
  - If no active feature (orchestrator, downtime, maintenance): allow (do not break normal non-feature sessions) but log or soft-warn.
  - Fail-closed: on any misconfig/ambiguous, block the edit and emit clear message with the violating path + active feature.
- Contract with builder briefs: every BUILD brief must set CLAUDE_ACTIVE_FEATURE=F-XXXX before any edits; subs must not override.
- Glob engine: support ** / * simple patterns as used in features (e.g. "src/**", ".claude/**").
- Integration: wired in .claude/settings.json hooks; also surfaces in scripts/verify.sh (contract tests) and assertion-shield.
- No impact on read-only tools or non-edit actions.
- Tests: cover block (outside authz or in forbidden), allow (inside), no-active-feature safe case. At least the 3 ACs exercised.
