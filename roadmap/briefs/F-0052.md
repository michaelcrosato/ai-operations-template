# Brief: F-0052 — oneshot hardening: admission metacharacters + verdict↔admit coupling (Tier B)

Immutable brief. Builder: `builder`. Branch: `feat/F-0052` (already checked out). Do not rescope; escalate blockers in your report.

## Why (context)
The 2026-07-18 review wave (DECISIONS 2026-07-18) verified: `src/oneshot/admit.js`'s `CHAINING_RE = /[\n;&|]/` blocks only four characters — command substitution (dollar-paren and backtick), input/output redirection (greater-than and less-than), and carriage returns are all ADMITTED and then executed by `src/oneshot/verdict.js` under `spawnSync {shell: true}` with no timeout; `verdict()` never re-checks admission; a PASS returns no evidence path; evidence writes are non-atomic with millisecond-timestamp names. F-0047 deliberately scoped to chaining only. Deliberate NON-goals (recorded in DECISIONS 2026-07-18 — do not build): env/cwd sandboxing, moving the tmpdir default, replacing `shell: true` (the acceptance command is the supervising operator's own command, now admission-screened).

## Acceptance (verbatim from the feature row)
1. admit rejects an acceptanceCommand containing command substitution (dollar-paren or backtick), output or input redirection (greater-than or less-than), or a carriage return, with reason no-verifiable-criterion - at least one test per metacharacter class
2. verdict re-runs admit on its descriptor first and returns a distinct NOT-ADMITTED outcome without spawning anything when admission fails - a test feeds a chained command directly to verdict
3. the acceptance-command spawn gets a configurable timeout with a documented default; an over-time command yields NOT-DONE with the evidence file recording the timeout - deterministic test
4. a PASS result includes the evidencePath and a test asserts the returned file exists and records exit 0
5. evidence writes are atomic (temp file then rename) with a collision-resistant filename (process id plus randomness, not bare milliseconds); all F-0047 pinned behaviors still pass unmodified
6. bash scripts/verify.sh exits 0 with evidence under roadmap/evidence/F-0052/

## Design notes (follow; escalate deviations)
- **admit.js:** extend the rejection surface beyond `CHAINING_RE` — reject `$(`, backtick, `>`, `<`, and `\r` (the existing `\n` stays). Keep the same rejection reason string `no-verifiable-criterion` and the existing code structure/doc-comment style (update the comment to name the new classes and cite F-0052). Note: reject the two-character sequence `$(` rather than bare `$` (env-var references like `$HOME` in a command remain admissible; substitution is the execution vector). Bare `$` followed by anything else stays admitted — say so in the comment.
- **verdict.js:** require `admit.js` and call `admit(descriptor, options)` first. If REJECT → return a distinct outcome (e.g. `{ verdict: 'NOT-ADMITTED', reason: <admit's reason> }`) WITHOUT spawning and WITHOUT writing evidence. Keep the existing NOT-DONE/PASS contract otherwise.
- **Timeout:** add `timeout` to spawnSync options — configurable via `options.timeoutMs` with a documented default (pick 10 minutes = 600000 ms, documented in the typedef like the F-0040 budget pattern). On timeout, spawnSync returns with `error` (ETIMEDOUT) — map to NOT-DONE and record `timedOut: true` + the limit in the evidence JSON. Deterministic test: tiny timeout (e.g. 200ms) + a `node -e` sleep command.
- **PASS payload:** `{ verdict: 'PASS', evidencePath }` (keep NOT-DONE's existing fields; adding evidencePath there too is fine and symmetric).
- **Atomic evidence write:** write to `<final>.<pid>.<randomhex>.tmp` in the SAME directory then `fs.renameSync`. Filename: replace bare `Date.now()` with `oneshot-verdict-<Date.now()>-<pid>-<randomhex>.json` (keep it sortable). Fix the stale "timestamped to avoid collisions" comment.
- **F-0047 pinned behaviors must still pass unmodified:** spawn-error→exitCode 127 NOT-DONE; >16MB maxBuffer; write-failure throws (missing evidence dir must still throw — atomic write must not silently create directories); stdout/stderr recorded; budget boundary tests; all existing admit rejections.

## Tests

`src/oneshot/verdict.test.js`: NOT-ADMITTED path (chained command fed straight to verdict — nothing spawned, no evidence file created); timeout → NOT-DONE with `timedOut: true`; PASS returns existing `evidencePath` whose JSON records exit 0; atomicity (no `*.tmp` residue next to the evidence file after a run).
