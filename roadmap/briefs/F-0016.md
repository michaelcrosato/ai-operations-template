### Fix (c) — Assertion shield: detect a missing upstream, print one calm line (`scripts/assertion-shield.ts`)

The current `getGitDiff()` unconditionally runs `git diff origin/develop...HEAD`; on a first commit (no `origin/develop`) git prints `fatal: ambiguous argument 'origin/develop...HEAD': unknown revision...` to stderr, which leaks to the console even though the catch handles control flow. Fix it by **checking the ref exists before diffing it**, so the failing command is never run. Replace the existing `getGitDiff()` function (lines ~11–34, from `function getGitDiff(): string {` through its closing `}`) with:

```ts
function refExists(ref: string): boolean {
  // --verify --quiet resolves the ref without printing anything; stdio 'pipe'
  // keeps git's stderr off the console so a missing upstream is silent.
  try {
    execFileSync('git', ['rev-parse', '--verify', '--quiet', ref], { stdio: 'pipe' });
    return true;
  } catch {
    return false;
  }
}

function getGitDiff(): string {
  // execFileSync with arg arrays: BASE_BRANCH is env-supplied, so it must
  // never pass through a shell (security-guidance plugin finding).
  const diffs: string[] = [];
  if (refExists(BASE_BRANCH)) {
    // Committed work on this branch vs base
    try {
      diffs.push(execFileSync('git', ['diff', `${BASE_BRANCH}...HEAD`], { encoding: 'utf8' }));
    } catch {
      // base resolved but the range diff failed — fall through to staged check
    }
  } else if (refExists('HEAD~1')) {
    // base not fetched/available but prior history exists — diff the last commit
    try {
      diffs.push(execFileSync('git', ['diff', 'HEAD~1'], { encoding: 'utf8' }));
    } catch {
      // no usable history — staged check below may still apply
    }
  } else {
    // First commit before the upstream base exists yet — not an error, just no
    // base to diff against. The staged check below still guards this commit.
    console.log(`[Assertion Shield] No '${BASE_BRANCH}' upstream and no prior commit yet (first commit) — auditing staged changes only.`);
  }
  try {
    // Staged-but-uncommitted changes — what a pre-commit hook is actually
    // gating. Without --cached the hook only ever saw prior commits.
    diffs.push(execFileSync('git', ['diff', '--cached'], { encoding: 'utf8' }));
  } catch {
    console.log('Not in a git repository. Skipping assertion check.');
  }
  return diffs.join('\n');
}
```

Constraints:
- The calm message MUST contain the literal substring `first commit` (a contract test greps for it) and MUST NOT contain the word `fatal`.
- Do not change `scanDiffForWeakening`, `run()`, the `BASE_BRANCH` constant, or any other behavior. The existing shield contract tests set `BASE_BRANCH=base` (a ref that DOES exist in their fixture), so they must keep passing unchanged — `refExists('base')` is true → the `base...HEAD` diff still runs.
- `assertion-shield.ts` must stay biome-clean (verify.sh runs `biome lint scripts`). Each `catch` block carries a comment (non-empty), matching the existing idiom — keep it that way.
