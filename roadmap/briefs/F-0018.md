# Builder Brief — F-0018: ForgeOps Visual Canvas — ReactFlow graph editor, live A/B sim, RBAC modes, exports

> Immutable. Build exactly this; report back; nothing more.

## Feature

Interactive visual canvas for the ForgeOps agent platform. Users author graphs via prompt or direct edit (ReactFlow), run A/B simulations with live structured logs, toggle RBAC personas (owner/viewer), export to portable JSON + Docker. Depends on F-0017 core foundations. Applies frontend.md rules strictly. Product-only paths.

## Spec

- Canvas: @xyflow/react powered infinite canvas rendering nodes (id/type/label/model/prompt/position/estimatedCost/etc) + edges (source/target). Supports drag, connect, delete, inline or panel edit of node fields. Prompt input area: natural language -> F-0017 promptToGraph -> immediate render of executable graph.

- Simulation controls: "Run A/B", step, pause. Delegates to F-0017 abSim + generateGraphAwareLog. Real-time log stream/ticker (timestamped entries with nodeId, message, level, costDelta, tokens, latencyMs). Optional Recharts viz for deltas.

- RBAC surface: explicit mode toggle or persona selector (owner vs viewer). Owner: full canvas edit + run + export + intervene. Viewer: read-only graph view + logs; any mutation attempt shows clear 403/404 feedback and is rejected (wired to F-0017 rbac.js simulation). No trust of client IDs.

- Exports: toolbar actions "Export JSON", "Export Docker Scaffold", "Copy for runExportedGraph". Uses F-0017 export.js to produce the graph.json + minimal Dockerfile + docker-compose.yml (DEMO_MODE aware, healthchecks). Preview + download.

- State + UX: every path has explicit handling — loading (skeleton canvas), empty (prompt hint + sample seed button), error (plain English message + retry), success (full interactive). Follows existing app/demo, components/ui, theme patterns exactly. a11y: aria, labels on all controls, keyboard reachable (no mouse-only canvas ops if possible).

- Integration only: consumes src/forge/* (no modifications to core). Leverages prior demo/seed.ts patterns for determinism. Matches package.json React 19 / Next 15 setup.

## Acceptance criteria (verbatim from roadmap/features.json F-0018)

- Canvas renders and allows interactive editing of graphs produced by F-0017 promptToGraph using ReactFlow (@xyflow/react)
- A/B simulation runs from canvas, streams real-time structured logs (from abSim/generateGraphAwareLog) to UI ticker/chart
- RBAC persona toggle enforces owner (full edit/run/export) vs viewer (read-only, mutations rejected visibly)
- Export actions produce valid JSON + Dockerfile/docker-compose from F-0017 export; downloadable and match spec
- UI handles all states (loading/empty/error/success) with plain messages, follows frontend.md and existing demo patterns, verify green
