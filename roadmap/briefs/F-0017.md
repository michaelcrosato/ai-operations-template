## Spec
Core non-UI foundations for the ForgeOps agent operations platform:
- promptToGraph (or equivalent pure function / CLI entry in src/ or lib/): takes natural language prompt and produces valid executable agent graph JSON (nodes with id/type/label/position/model/prompt/estimatedCost/etc + edges source/target). Must be deterministic enough for downstream execution and canvas rendering.
- Basic A/B simulation engine: execute two graph variants over synthetic inputs/datasets, emit structured real-time logs (ts, nodeId, message, level, costDelta, tokens, latencyMs) consumable by CLI or UI hook/ticker.
- RBAC simulation layer (for demo/product surface): owner (full edit/run/intervene/publish), non-owner (read-only; 403/404 on mutation attempts). Use persona or token seam matching existing demo patterns.
- Exports: portable JSON graph + minimal multi-stage Dockerfile + docker-compose (with DEMO_MODE, healthcheck) that runs the graph using the existing engine stdlib/walker. Clean integration (no engine source changes).
- Real-time logs + metrics hooks that the (future) canvas can consume.
All under product paths only. No touches to .claude/, scripts/ (except allowed package.json for runtime deps if any), roadmap/, or engine core. Leverage existing lib/seed.ts patterns, cloneSeed, generateGraphAwareLog, etc. where helpful for determinism. Add units + product-mode verify coverage.
