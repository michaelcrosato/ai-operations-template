# Builder Brief — F-0014: E2E workflow lane parity — e2e.yml must satisfy the gate it runs

> Immutable. Build exactly this; report back; nothing more. (Orchestrator: explorer fan-out skipped — context gathered directly: both adopter fix patches fetched from GitHub, ci.yml / e2e.yml / verify.sh read; everything the builder needs is embedded below.)

## Feature

Second field defect, hit independently by both adopters. `.github/workflows/e2e.yml` runs `bash scripts/verify.sh --e2e`, and verify.sh **hard-fails when `CI=true` and actionlint is absent** — but e2e.yml never installs actionlint, so the E2E lane breaks the moment an adopter defines an `e2e` script. The lane also diverges from ci.yml on shallow checkout (assertion-shield diffs against `origin/develop`, needs `fetch-depth: 0`) and Node version (20 vs 24 — that divergence already bit once, DECISIONS 2026-06-10), and its `paths` filter omits both e2e.yml itself and other verify.sh inputs. Spec source: department brief + the two proven adopter fixes (agy-software-2 PR #18; claude-software-3 commit d234838) — this is a port of their union, plus a lane-parity contract test so workflows invoking verify.sh cannot silently diverge again. Spec_ref: roadmap/DECISIONS.md (entry lands at RECORD).

### 1. Replace `.github/workflows/e2e.yml` with exactly this content

```yaml
name: E2E

on:
  workflow_dispatch:
  pull_request:
    paths:
      [
        "src/**",
        "tests/**",
        "e2e/**",
        "playwright.config.*",
        "cypress.config.*",
        "package.json",
        "package-lock.json",
        ".github/workflows/e2e.yml",
      ]

concurrency:
  group: e2e-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  e2e:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0 # assertion-shield diffs against origin/develop (same as ci.yml)

      - uses: actions/setup-node@v6
        with:
          # 24 matches ci.yml and the development environment — this job runs the
          # same verify.sh gate, so the runtimes must not diverge (DECISIONS 2026-06-10)
          node-version: 24
          cache: npm

      - name: Install
        run: npm ci --no-audit --no-fund

      - name: Install actionlint (pinned v1.7.12, checksum-verified)
        run: |
          mkdir -p "$HOME/.local/bin"
          curl -sSfL https://github.com/rhysd/actionlint/releases/download/v1.7.12/actionlint_1.7.12_linux_amd64.tar.gz -o /tmp/actionlint.tar.gz
          echo "8aca8db96f1b94770f1b0d72b6dddcb1ebb8123cb3712530b08cc387b349a3d8  /tmp/actionlint.tar.gz" | sha256sum -c -
          tar -C "$HOME/.local/bin" -xzf /tmp/actionlint.tar.gz actionlint
          echo "$HOME/.local/bin" >> "$GITHUB_PATH"

      - name: E2E suite (skips with notice until the product defines an e2e script)
        run: |
          if node -e "process.exit(require('./package.json').scripts?.e2e ? 0 : 1)"; then
            bash scripts/verify.sh --e2e
          else
            echo "No e2e script defined yet (pre-Phase-3). Skipping."
          fi
```

The actionlint install block must stay **byte-identical** to the one in ci.yml (same version, same sha256, same step name) — the contract test below derives its reference values from ci.yml. Do not modify ci.yml.

### 2. Append a lane-parity contract-test section to `scripts/test-hooks.sh`

Place it after the existing sections, before the final PASS/FAIL summary, following the file's existing idiom (`check <name> <expected-exit> <actual-exit>`, `echo "── section"` headers, the harness's existing tmp/fixture-dir conventions). Shape (adapt variable names/tmp paths to the harness idiom — behavior is the contract):

```bash
echo "── workflow lane parity (F-0014)"
# Any workflow that invokes verify.sh must satisfy the gate it runs: verify.sh
# hard-requires actionlint under CI, assertion-shield needs full history
# (fetch-depth 0), and Node 20/24 divergence already bit once (DECISIONS
# 2026-06-10). Reference values come from ci.yml so a future pin bump cannot
# silently diverge the lanes.
WF_DIR="$ROOT/.github/workflows"
REF_NODE="$(grep -Eo 'node-version: *[0-9]+' "$WF_DIR/ci.yml" | head -1 | grep -Eo '[0-9]+')"
REF_PIN="$(grep -Eo 'actionlint/releases/download/v[0-9][0-9.]*' "$WF_DIR/ci.yml" | head -1)"
REF_SHA="$(grep -Eo '[0-9a-f]{64}' "$WF_DIR/ci.yml" | head -1)"

# lane_parity <workflows-dir>: exit 0 iff every verify.sh-invoking workflow in
# the dir carries ci.yml's pinned actionlint (version+sha), fetch-depth 0, and
# ci.yml's node-version.
lane_parity() {
  local dir="$1" wf bad=0
  for wf in "$dir"/*.yml; do
    [ -f "$wf" ] || continue
    grep -q 'verify\.sh' "$wf" || continue
    grep -qF "$REF_PIN" "$wf" || bad=1
    grep -qF "$REF_SHA" "$wf" || bad=1
    grep -Eq 'fetch-depth: *0' "$wf" || bad=1
    grep -Eq "node-version: *${REF_NODE}([^0-9]|$)" "$wf" || bad=1
  done
  return "$bad"
}

check "lane parity: every verify.sh workflow conforms" 0 "$(lane_parity "$WF_DIR"; echo $?)"
check "lane parity: refs extracted from ci.yml" 0 "$(if [ -n "$REF_NODE" ] && [ -n "$REF_PIN" ] && [ -n "$REF_SHA" ]; then echo 0; else echo 1; fi)"

# negative: a fixture workflow running verify.sh on Node 20 with shallow
# checkout and no actionlint must FAIL the parity check
<fixture dir per harness convention>/broken.yml  ← write via cat-heredoc:
  name: broken
  on: workflow_dispatch
  jobs:
    e2e:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v6
        - uses: actions/setup-node@v6
          with:
            node-version: 20
        - run: bash scripts/verify.sh
check "lane parity: broken fixture workflow rejected" 1 "$(lane_parity "<fixture-dir>"; echo $?)"

check "e2e.yml triggers on its own changes" 0 "$(grep -qF '.github/workflows/e2e.yml' "$WF_DIR/e2e.yml" && echo 0 || echo 1)"
```

Notes:
- ci.yml itself passes `lane_parity` by construction (it is its own reference); e2e.yml passes only after your edit — that is the point.
- The fixture must live under the harness's existing temp/fixture root (cleaned up like the other fixtures), **never** inside the real `.github/workflows/`.
- Do not weaken, skip, or reorder any existing test.

## Acceptance criteria (verbatim from roadmap/features.json)

1. e2e.yml installs actionlint before invoking verify.sh, using the identical pinned, checksum-verified step as ci.yml (same version, same sha256)
2. e2e.yml checkout uses fetch-depth 0 so the assertion shield can diff against origin/develop
3. e2e.yml pins node-version 24, matching ci.yml and the development environment
4. The e2e.yml pull_request paths filter includes .github/workflows/e2e.yml itself plus tests/**, package.json and package-lock.json alongside the existing src/e2e/config entries
5. Lane-parity contract tests in scripts/test-hooks.sh fail when a workflow that invokes verify.sh lacks the pinned actionlint install used by ci.yml, lacks fetch-depth 0, or pins a node-version different from ci.yml (negative-tested against a broken fixture workflow), and pass against the real .github/workflows
6. bash scripts/verify.sh exits 0
