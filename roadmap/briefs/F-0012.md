# Builder Brief — F-0012: seed.ts delegating shim — E2E gate must not pass unseeded in product mode

> Immutable. Build exactly this; report back; nothing more. (Orchestrator: explorer fan-out skipped — surface is small and the orchestrator read `scripts/seed.ts`, `scripts/verify.sh`, `scripts/test-hooks.sh`, and `package.json` directly; all relevant excerpts are inlined below.)

## Feature

Adoption trial #1 (adopter repo `agy-software-2`) surfaced an engine defect: `scripts/verify.sh --e2e` hard-codes `npx ts-node scripts/seed.ts` (verify.sh line 123), and the template stub exits 0 unconditionally after its prod-refusal guard. In an adopter repo with product code but no seeding defined, the E2E gate therefore goes green against an **unseeded database**. Fix: rewrite the stub as a delegating shim that (a) keeps the prod guard, (b) delegates to the product's `package.json` `"seed"` script when one exists, (c) hard-fails in product mode when none exists, (d) keeps exit-0 stub behavior in template mode. A reference fix was verified green in the adopter; this is its template-idiom adaptation. `spec_ref: roadmap/DECISIONS.md` (department defect report, 2026-06-11 entry — written at RECORD).

## Spec

### 1. Rewrite `scripts/seed.ts` as a delegating shim

Replace the file body with exactly this behavior, in this order (cwd-relative paths are intentional — verify.sh `cd`s to repo root before invoking, and contract tests `cd` into fixture dirs):

1. **Prod-refusal guard (unchanged, first):** if `process.env.DATABASE_URL ?? ''` matches `/prod/i`, print `[seed] REFUSING: connection string looks like production.` to stderr and `process.exit(1)`.
2. **Delegation:** read `package.json` from cwd (`fs.readFileSync` + `JSON.parse`, wrapped in try/catch → treat unreadable/missing as "no seed script"). If `scripts.seed` is a non-empty string:
   - **Recursion sentinel:** if `process.env.SEED_SHIM_ACTIVE === '1'`, print `[seed] REFUSING: circular delegation — package.json "seed" script re-invokes scripts/seed.ts.` to stderr and `process.exit(1)`. (Without this, an adopter wiring `"seed": "ts-node scripts/seed.ts"` would loop forever inside CI.)
   - Otherwise spawn the child and exit with its status:
     ```ts
     const res = spawnSync('npm', ['run', '--silent', 'seed'], {
       stdio: 'inherit',
       shell: process.platform === 'win32',   // npm is npm.cmd on Windows
       env: { ...process.env, SEED_SHIM_ACTIVE: '1' },
     });
     process.exit(res.status ?? 1);           // signal-killed / spawn error → fail
     ```
3. **Product mode, no seed script:** if `fs.existsSync('src')`, print to stderr: `[seed] FAILED: product code exists but package.json defines no "seed" script — the E2E gate must not pass against an unseeded database.` and `process.exit(1)`.
4. **Template mode:** log `[seed] template stub — no product schema yet. Define a "seed" script in package.json when the data layer lands (Phase 2).` and `process.exit(0)`.

Keep a header comment in the existing style (see current file): state the four rules that survive any stack (synthetic data only · deterministic · idempotent · refuses production) — they now describe the contract the delegated-to product seed script must honor — plus one line explaining the shim's four-branch contract. Use `node:` -prefixed imports (`node:child_process`, `node:fs`) — biome lints `scripts/`.

### 2. Add a contract-test section to `scripts/test-hooks.sh`

Insert a new section after the `assertion-shield.ts` section (which defines `TSNODE="$ROOT/node_modules/ts-node/dist/bin.js"` — reuse it; if you place the section elsewhere, define the variable locally). Use the existing `check <name> <expected> <actual>` helper and fixture-dir style:

```bash
echo "── seed.ts delegating shim"
SD="$(mktemp -d)"
printf '{ "name": "fixture", "scripts": {} }\n' > "$SD/package.json"
check "template mode (no src, no seed script) exits 0" 0 "$(cd "$SD" && node "$TSNODE" "$ROOT/scripts/seed.ts" >/dev/null 2>&1; echo $?)"
check "refuses prod DATABASE_URL" 1 "$(cd "$SD" && DATABASE_URL=postgres://u@prod-db/x node "$TSNODE" "$ROOT/scripts/seed.ts" >/dev/null 2>&1; echo $?)"
mkdir -p "$SD/src"
check "product mode without seed script fails" 1 "$(cd "$SD" && node "$TSNODE" "$ROOT/scripts/seed.ts" >/dev/null 2>&1; echo $?)"
printf '{ "name": "fixture", "scripts": { "seed": "node -e \\"process.exit(0)\\"" } }\n' > "$SD/package.json"
check "delegates to seed script (success)" 0 "$(cd "$SD" && node "$TSNODE" "$ROOT/scripts/seed.ts" >/dev/null 2>&1; echo $?)"
printf '{ "name": "fixture", "scripts": { "seed": "node -e \\"process.exit(3)\\"" } }\n' > "$SD/package.json"
check "propagates seed script failure" nonzero "$(cd "$SD" && node "$TSNODE" "$ROOT/scripts/seed.ts" >/dev/null 2>&1; rc=$?; [ "$rc" -ne 0 ] && echo nonzero || echo zero)"
check "circular delegation fails fast" 1 "$(cd "$SD" && SEED_SHIM_ACTIVE=1 node "$TSNODE" "$ROOT/scripts/seed.ts" >/dev/null 2>&1; echo $?)"
rm -rf "$SD"
```

Notes you must respect:
- The failure-propagation check normalizes to `nonzero`/`zero` deliberately — do **not** assert the literal exit code 3 (npm version differences); do not weaken it to expect 0.
- The delegation-success and propagation fixtures require `npm` on PATH inside the fixture cwd — that holds (CI and dev both have node+npm). The fixture has no `node_modules`; the seed scripts above use only `node -e`, which is why they're shaped that way. Keep them that way.
- `shellcheck` runs over `scripts/*.sh` in verify.sh — your additions must be shellcheck-clean.
- Adjust the final `echo`/count lines not at all — `PASS`/`FAIL` counters come from `check`.

## Acceptance criteria (verbatim from roadmap/features.json)

1. seed.ts keeps the prod-refusal guard: exits 1 when DATABASE_URL matches /prod/i, before any delegation
2. When package.json defines a seed script, seed.ts runs npm run --silent seed and exits with the child exit status (success and failure both propagate)
3. Product mode (src/ exists) with no seed script defined exits 1 with a clear message - the E2E gate cannot pass unseeded
4. Template mode (no src/, no seed script) logs the stub message and exits 0
5. Circular delegation (a seed script that re-invokes scripts/seed.ts) fails fast with a clear error instead of looping
6. Contract tests in scripts/test-hooks.sh cover the five behaviors above; bash scripts/verify.sh exits 0
