## Feature

CLAUDE.md §6 carries a hard rule: *"Never end a session while a PR you opened has CI pending — watch `gh pr checks <n> --watch` to completion, or write a HANDOFF line."* Three times across the portfolio a session has exited while "watching CI," the watcher died with the process, and a green-but-unmerged PR was stranded overnight. This feature mechanizes that rule into one battle-tested tool: `scripts/ship.sh <pr#> [--merge]`. It blocks on the PR's checks until they reach a terminal state, treats **only** an all-green result as success, and — when `--merge` is given — merges only then. Its key hardening (claude-software-3 PR #17): it waits a **bounded** time for checks to *register* after PR creation and **fails closed** (refuses to merge) if none ever appear, closing the race where `gh pr checks --watch` returns instantly against a PR that has no checks yet and falsely reads "green." Spec_ref: CLAUDE.md §6; roadmap/DECISIONS.md (entry lands at RECORD).

## Spec

### 1. Create `scripts/ship.sh`

A POSIX bash script, `set -eu`, portable to git-bash on Windows + ubuntu CI (no GNU-only flags), **shellcheck-clean** (verify.sh runs `shellcheck scripts/*.sh` and it is a hard gate). Match the house idiom of `scripts/install-into.sh` (header comment, `echo` progress, `>&2` for errors).

**Usage:** `bash scripts/ship.sh <pr-number> [--merge]`

**Behavior, in order:**

1. **Parse args.** Exactly one PR number (a string of digits) is required; `--merge` is the only allowed flag and may appear before or after the number. On a missing PR number, a non-numeric PR number, or any unknown flag: print a one-line usage message to stderr and `exit 1`.

2. **Require `gh`.** If `command -v gh` fails, print a clear error to stderr and `exit 1`.

3. **Base-branch guard (only when `--merge`).** Read the PR's base branch:
   `gh pr view "$PR" --json baseRefName --jq .baseRefName`. If it is `main` or `master`, refuse: print an error and `exit 1` **without merging** (mirrors §6 "never push master/main"). Do this *before* the wait so it fails fast. If `--merge` is not set, skip this.

4. **Bounded register-wait.** Poll `gh pr checks "$PR"` (capture stdout+stderr together) until checks have registered, or until a timeout. Detection rule — treat the PR as **not yet registered** iff the captured output matches, case-insensitively, `no checks` (gh prints `no checks reported on the '<branch>' branch` when none exist yet); otherwise treat checks as registered (pending/pass/fail all count as registered). Loop: at most `SHIP_REGISTER_TIMEOUT` seconds total, sleeping `SHIP_REGISTER_INTERVAL` seconds between probes. **Defaults:** `SHIP_REGISTER_TIMEOUT=180`, `SHIP_REGISTER_INTERVAL=6`. If the timeout elapses with checks still unregistered: print
   `No checks registered after <N>s — refusing to merge an unchecked PR (fail closed).`
   to stderr and `exit 1`. **No merge is ever attempted on this path.**

5. **Watch to terminal.** Run `gh pr checks "$PR" --watch` and capture its exit code. `gh pr checks --watch` blocks until every check completes and exits 0 **iff** all passed. If its exit code is non-zero (a failing or otherwise non-green result), print a message and `exit` with that non-zero code. **No merge on this path.**

6. **Merge or report.**
   - If `--merge` was given and we reached here (all green): `gh pr merge "$PR" --merge`. Propagate its exit code — if the merge command fails, ship.sh exits non-zero.
   - If `--merge` was *not* given: print that checks are green and that `--merge` was not requested, and `exit 0` **without merging**.

**Environment knobs** (`SHIP_REGISTER_TIMEOUT`, `SHIP_REGISTER_INTERVAL`) are *timing tuning only*, not safety overrides, so they do **not** need `DANGEROUSLY_` naming: a smaller/zero timeout makes the tool fail closed *sooner* (never merges an unchecked PR), a larger one only waits longer — neither can bypass the green-only-merge invariant. Put a one-line comment in the script saying exactly this (a security-review reviewer will look for it).

Code-shape sketch (adapt freely; behavior is the contract, not these exact lines):

```bash
#!/usr/bin/env bash
# ship.sh — watch a PR's checks to terminal, merge ONLY on green (CLAUDE.md §6).
# Hardening: bounded wait for checks to REGISTER post-creation; fail CLOSED if
# none appear (never merge an unchecked PR). Usage: bash scripts/ship.sh <pr#> [--merge]
set -eu

PR=""
MERGE=false
for arg in "$@"; do
  case "$arg" in
    --merge) MERGE=true ;;
    --*) echo "ERROR: unknown flag '$arg'" >&2; exit 1 ;;
    *) if [ -z "$PR" ]; then PR="$arg"; else echo "ERROR: unexpected argument '$arg'" >&2; exit 1; fi ;;
  esac
done
case "$PR" in
  ''|*[!0-9]*) echo "ERROR: a numeric PR number is required. Usage: bash scripts/ship.sh <pr#> [--merge]" >&2; exit 1 ;;
esac
command -v gh >/dev/null 2>&1 || { echo "ERROR: gh (GitHub CLI) not found on PATH." >&2; exit 1; }

# SHIP_REGISTER_TIMEOUT / SHIP_REGISTER_INTERVAL are timing knobs ONLY — a smaller
# value fails closed sooner, never opens a bypass, so no DANGEROUSLY_ prefix.
TIMEOUT="${SHIP_REGISTER_TIMEOUT:-180}"
INTERVAL="${SHIP_REGISTER_INTERVAL:-6}"

if [ "$MERGE" = true ]; then
  BASE="$(gh pr view "$PR" --json baseRefName --jq .baseRefName 2>/dev/null || echo '')"
  case "$BASE" in
    main|master) echo "ERROR: PR #$PR targets '$BASE' — refusing to merge (never master/main, §6)." >&2; exit 1 ;;
  esac
fi

elapsed=0
registered=false
while :; do
  probe="$(gh pr checks "$PR" 2>&1 || true)"
  if printf '%s' "$probe" | grep -qi 'no checks'; then
    if [ "$elapsed" -ge "$TIMEOUT" ]; then
      echo "No checks registered after ${TIMEOUT}s — refusing to merge an unchecked PR (fail closed)." >&2
      exit 1
    fi
    sleep "$INTERVAL"; elapsed=$((elapsed + INTERVAL)); continue
  fi
  registered=true; break
done
[ "$registered" = true ] || { echo "fail closed" >&2; exit 1; }

if gh pr checks "$PR" --watch; then
  if [ "$MERGE" = true ]; then
    gh pr merge "$PR" --merge
  else
    echo "PR #$PR is green. --merge not requested; not merging."
  fi
else
  rc=$?
  echo "PR #$PR checks are not green — not merging." >&2
  exit "$rc"
fi
```

Note the `set -eu` interaction: `gh pr checks "$PR" --watch` is used as the condition of an `if`, so a non-zero exit there does **not** trigger `set -e` — that is intentional. Capture `$?` inside the `else` branch as shown. Keep shellcheck happy (quote everything; `rc=$?` immediately).

### 2. Append a `ship.sh` contract-test section to `scripts/test-hooks.sh`

Place it **after** the existing `── workflow lane parity (F-0014)` section and **before** the final `echo ""` / summary block. Follow the file's idiom exactly: `echo "── ship.sh"` header, the `check <name> <expected-exit> <actual-exit>` helper, `mktemp -d` fixtures cleaned up at the end, `set -u`-safe (the file runs under `set -u`, not `set -e`).

The tests must run with **no network and no real `gh`** by putting a **stub `gh`** first on `PATH`. The stub is driven by env vars so each scenario is deterministic and fast. Write it via heredoc, `chmod +x`. Required stub behavior:

```bash
SHIP_FIX="$(mktemp -d)"
STUB="$SHIP_FIX/bin"; mkdir -p "$STUB"
cat > "$STUB/gh" <<'GHSTUB'
#!/usr/bin/env bash
# Fake gh for ship.sh contract tests. Driven by env:
#   FAKE_NOCHECK_FILE  — path to a file holding an integer: how many more
#                        `pr checks` probes should report "no checks" before
#                        checks "register". Decremented each probe.
#   FAKE_WATCH_RC      — exit code for `pr checks <pr> --watch` (default 0).
#   FAKE_BASE          — base branch reported by `pr view` (default develop).
#   FAKE_MERGE_MARKER  — file touched when `pr merge` is invoked.
#   FAKE_MERGE_RC      — exit code for `pr merge` (default 0).
sub="${1:-} ${2:-}"
case "$*" in
  *"--watch"*) echo "watch: done"; exit "${FAKE_WATCH_RC:-0}" ;;
esac
case "$sub" in
  "pr checks")
    n=0; [ -n "${FAKE_NOCHECK_FILE:-}" ] && n="$(cat "$FAKE_NOCHECK_FILE" 2>/dev/null || echo 0)"
    if [ "$n" -gt 0 ]; then
      echo $((n - 1)) > "$FAKE_NOCHECK_FILE"
      echo "no checks reported on the 'feat/x' branch" >&2
      exit 1
    fi
    printf 'CI\tpass\t1s\thttps://x\n'; exit 0 ;;
  "pr view") echo "${FAKE_BASE:-develop}"; exit 0 ;;
  "pr merge") [ -n "${FAKE_MERGE_MARKER:-}" ] && : > "$FAKE_MERGE_MARKER"; exit "${FAKE_MERGE_RC:-0}" ;;
esac
exit 0
GHSTUB
chmod +x "$STUB/gh"
```

Then a helper to invoke ship.sh with the stub on PATH and fast timing, e.g.:

```bash
SHIP="$ROOT/scripts/ship.sh"
MARK="$SHIP_FIX/merged"
run_ship() { # run_ship <args...> ; echoes exit code. Per-test env set by caller via env vars already exported.
  ( PATH="$STUB:$PATH" SHIP_REGISTER_TIMEOUT="${SHIP_REGISTER_TIMEOUT:-3}" SHIP_REGISTER_INTERVAL="${SHIP_REGISTER_INTERVAL:-1}" \
      FAKE_MERGE_MARKER="$MARK" bash "$SHIP" "$@" >/dev/null 2>&1; echo $? )
}
merged() { if [ -f "$MARK" ]; then echo yes; else echo no; fi; }
```

Reset `$MARK` (`rm -f "$MARK"`) and set the scenario env (write `FAKE_NOCHECK_FILE`, export `FAKE_WATCH_RC`/`FAKE_BASE`/`FAKE_MERGE_RC`) before each ship.sh call. **At least these checks** (you may add more; keep them all):

1. `check "ship: rejects missing PR number" 1 "$(run_ship)"`
2. `check "ship: rejects non-numeric PR" 1 "$(run_ship abc)"`
3. `check "ship: rejects unknown flag" 1 "$(run_ship 5 --bogus)"`
4. No-checks-ever → fail closed: write a large number (e.g. `9999`) to `FAKE_NOCHECK_FILE`, `SHIP_REGISTER_TIMEOUT=2`, `SHIP_REGISTER_INTERVAL=1`, `--merge` →
   `check "ship: fails closed when no checks ever register" 1 "$(run_ship 5 --merge)"`
5. …and it did **not** merge: `check "ship: no-checks path does not merge" no "$(merged)"`
6. Green + `--merge` merges: `FAKE_NOCHECK_FILE`→`0`, `FAKE_WATCH_RC=0`, reset marker →
   `check "ship: green + --merge merges" 0 "$(run_ship 5 --merge)"` then `check "ship: merge happened" yes "$(merged)"`
7. Green **without** `--merge` does not merge: reset marker →
   `check "ship: green without --merge exits 0" 0 "$(run_ship 5)"` then `check "ship: no merge without --merge" no "$(merged)"`
8. Failing checks → no merge: `FAKE_WATCH_RC=1`, reset marker, `--merge` →
   `check "ship: failing checks exit non-zero" nonzero "$(rc=$(run_ship 5 --merge); [ "$rc" -ne 0 ] && echo nonzero || echo zero)"` and `check "ship: failing checks do not merge" no "$(merged)"`
9. Delayed registration still proceeds: write `1` to `FAKE_NOCHECK_FILE` (one "no checks" then registered), `FAKE_WATCH_RC=0`, `SHIP_REGISTER_TIMEOUT=5`, reset marker, `--merge` →
   `check "ship: registers after a delay then merges" 0 "$(run_ship 5 --merge)"` and `check "ship: delayed registration merged" yes "$(merged)"`
10. Base main refused: `FAKE_BASE=main`, reset marker, `--merge` →
    `check "ship: refuses to merge PR based on main" 1 "$(run_ship 5 --merge)"` and `check "ship: main-based PR not merged" no "$(merged)"`
11. Merge-command failure propagates: registered + `FAKE_WATCH_RC=0` + `FAKE_MERGE_RC=1`, reset marker, `--merge` →
    `check "ship: merge failure propagates" nonzero "$(rc=$(run_ship 5 --merge); [ "$rc" -ne 0 ] && echo nonzero || echo zero)"`

Clean up: `rm -rf "$SHIP_FIX"` at the end of the section. Use the `nonzero`/`zero` pattern (as the existing `seed.ts` section does at lines 198) for any check where the exact non-zero code is not pinned. Do **not** weaken, skip, or reorder any existing test.

> Implementation caution for the `check` helper: it compares with `[ "$2" = "$3" ]`, so expected `1` matches only an exact `1`. For the fail-closed and base-refusal cases ship.sh exits exactly `1`, so use `1`. For the watch-failure and merge-failure cases the propagated code may differ, so use the `nonzero` sentinel pattern.
