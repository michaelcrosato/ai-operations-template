## Feature

The first two field installs of this engine (agy-software-2, fixed in its commit af92153; BayLine) both broke out of the box and had to hand-strip template self-proving artifacts from `package.json`: the `"test"` script points at `src/health.test.js` (adopters do not copy `src/`, so `verify.sh` fails immediately; copying `src/` instead would flip PRODUCT_MODE prematurely), and `"lint": "biome lint scripts src"` references the not-yet-existing `src` path (Biome internal-error noise). The proven adopter-clean shape from the field: **no `test` script, `"lint": "biome lint scripts"`**. This feature ships `scripts/install-into.sh` so the documented install procedure produces that shape automatically. Contract: **a fresh adopter passes `bash scripts/verify.sh` immediately after the documented procedure with zero hand-fixes.** Spec ref: `roadmap/DECISIONS.md` (department brief 2026-06-11, this entry will be appended at session RECORD).

## Spec

### 1. New file: `scripts/install-into.sh`

`bash scripts/install-into.sh <target-dir>` — run from the template repo root (resolve the template root from the script's own location like verify.sh does: `cd "$(dirname "$0")/.."`). Bash, `set -eu`, must pass shellcheck (the verify gate runs `npx --no-install shellcheck scripts/*.sh`). Portability bar: git-bash on Windows AND ubuntu CI — use `mktemp`, plain `cp -r`, no GNU-only flags.

**Refusals (check first, exit 1 with a clear message):**
- No `<target-dir>` argument, or target does not exist / is not a directory.
- Target resolves to the template root itself, or to an ancestor of the template root (`cp -r` into an ancestor risks recursive self-copy). Subdirectories of the template's `tmp/` are explicitly fine (scratch space for tests/demos).

**Never delete anything. Copy / create / append only. No network access.**

**Copy set — engine-owned, overwritten on every run (this is also the upgrade path):**
- Root docs: `CLAUDE.md`, `AGENTS.md`, `AI_OPERATIONS_PLAN.md`, `OPERATOR_GUIDE.md`
- `.claude/` (entire tree), `scripts/` (entire tree, includes this installer), `.github/` (entire tree)
- `docs/optional-modules.md` (README references it) — but NOT `docs/feedback-2-*` (template self-state)
- `roadmap/features.schema.json`, `roadmap/briefs/TEMPLATE.md` (structure, not state)

**Copy-if-absent, warn-if-present (target-owned once they exist):**
- `biome.json`, `tsconfig.json`, `.gitattributes` (when present in target, print a manual-merge warning; the `.gitattributes` LF pinning is load-bearing for the bash hooks)

**`.gitignore`:** if absent in target, copy the template's. If present, append any of these load-bearing lines that are missing (append at end — late negations win): `node_modules/`, `tmp/`, `*.log`, `!roadmap/evidence/**`, `.claude/settings.local.json`. The `!roadmap/evidence/**` negation is the critical one (evidence must ship).
